> 작성일: 2026-05-19
> 환경: cmux 내장 브라우저 (WKWebView), viewport 665×789 (iPad 폭), Dark 테마
> 대상 서버: `localhost:3001` (npm run dev / Next 16.0.7 Turbopack)
> 점검 대상 라우트: `/dashboard`, `/portfolio`, `/snapshots`, `/snapshots/[id]`, `/simulation`, `/accounts`, `/settings`, `/more` + AI 어시스턴트

**발견 사항**

### 🐛 H-1. 차트 토글 텍스트 세로 줄바꿈 (Critical)

`665px` 폭에서 `수익률 / 자산` + `1개월 / 3개월 / 6개월 / 1년 / 전체` 두 토글 그룹이 한 줄에 다 들어가지 않아, 각 라벨이 **글자 한 자씩 세로로 쪼개져** 표시됨.

- AS-IS: 「수」「익」「률」 / 「1」「개」「월」 같이 한 음절씩 분리 (스크린샷 `01-home-mobile.png` 참고)
- TO-BE: 라벨이 한 단위로 유지되도록 `whitespace-nowrap` + (필요 시) 가로 스크롤 또는 `text-[10px]` 같은 자동 축소

> 위치: 홈 차트 카드 우측 토글 영역
> 원인 추정: 두 토글 영역이 같은 `gap` flex row에 배치되어 있는데 각 버튼이 `text-[11px]` + `px-2.5`로 고정되어, viewport가 좁아지면 단어/문자 자동 줄바꿈이 발생

### ⚠️ H-2. 수익률 TOP 종목명 트렁케이션 (KIS 마스터 데이터 한계)

```
DIREX DAIL SEMI BU 3X ET-USD   ← SOXL (실제: DIREXION DAILY SEMICONDUCTOR BULL 3X)
PROSHARES ULTRA SEMICONDUCT    ← USD ticker (실제: PROSHARES ULTRA SEMICONDUCTORS)
```

KIS 종목 마스터의 영문명 길이 제한으로 잘린 채 저장된 것으로 보임. 표시 영역은 넓은데 데이터 자체가 절단되어 있음.

- 권장: Yahoo Finance 영문명을 fallback으로 사용 또는 `stocks.engName` 풀네임 별도 컬럼

### ⚠️ H-3. 토글 버튼 a11y 누락

- "수익률/자산", "1개월/3개월/…" 토글 버튼에 `aria-pressed` 또는 `role="tab" aria-selected` 없음
- 시각만으로는 활성 상태가 명확하나 스크린리더로는 인지 불가
- 참고: 보유 페이지의 계좌 탭(`[role=tab][aria-selected]`)은 정상 — 일관성 위해 동일 패턴 적용 권장

### 💡 H-4. `기간 변화: +11.15%p` 표기

최근 커밋(4b8ef7a)에서 일간·주간 변동의 `%p`를 제거했으나, 기간 변화는 그대로 `%p` 유지. 기간 변화는 수익률 간 "포인트 차이"라 정확한 표기이지만, 다른 영역과 표기가 어긋난다는 점은 인지 필요.

**발견 사항**

### ⚠️ P-1. 종목 검색 즉시 트리거 부재

종목 검색창에 "삼성"을 타이핑해도 결과가 즉시 표시되지 않고, **Enter 또는 우측 돋보기 버튼을 눌러야 검색이 실행됨**.

- 일반적인 자동완성 UX(타이핑 즉시 결과)와 다르므로 사용자가 "검색이 안 되네?"라고 오인할 가능성
- 권장: `onChange` debounce(300ms) → 즉시 결과 노출. 모바일 키보드 부담은 결과를 작은 카드로 줄이면 완화

### ⚠️ P-2. 정렬 토글 a11y (평가금/매입금/수익금)

```html
<button class="text-[11px] font-bold ... px-2 py-1 ...">평가금</button>
```

`aria-selected` 또는 `aria-sort` 없음. 현재 정렬 기준이 시각만으로 노출 → 스크린리더 미지원.

### ⚠️ P-3. FAB 두 개가 종목 카드 일부 가림

페이지 우하단에 `AI 어시스턴트` + `종목 추가` 두 개의 원형 FAB(`fixed right-4 z-40 w-12 h-12`)가 떠 있어, 좁은 폭에서 스크롤 시 종목 카드의 수익률/금액 영역과 겹침.

- 권장: 종목 카드의 우측 내부 패딩을 FAB 영역 폭만큼(약 64px) 확보하거나, FAB를 스크롤 시 자동 숨김(`opacity-0 translate-y-2`)

### ⚠️ P-4. 종목 영문명 트렁케이션 (홈과 동일)

종목 카드 상단 라벨도 동일하게 KIS 마스터 영문명이 잘린 채 표시됨 (`DIRX DLY TSLA BUL 2X ETF-AUI` 등). → H-2와 함께 처리.

### ⚠️ M-1. "시뮬레이션" vs "만약에" 차이 모호

같은 "과거 시점 기반 가정 분석" 도구가 두 개 노출되어 있음.

- 시뮬레이션: 과거 스냅샷 그대로 → 지금까지 보유했다면?
- 만약에: 과거 그 시점에 (다른 종목을) 샀더라면?

기능 차이는 분명하나, 라벨/설명만으로는 사용자가 처음에 헷갈릴 수 있음. 더 분명한 명칭/아이콘 차별화 권장 (예: "스냅샷 유지 시뮬레이션" / "대체 매수 시뮬레이션").
