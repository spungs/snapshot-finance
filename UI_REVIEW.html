P0 · BUGH-1. 차트 토글 텍스트가 글자 단위로 세로 줄바꿈

AS-IS · 665px 폭에서 「수」「익」「률」 / 「1」「개」「월」처럼 한 음절씩 분리 → 가독성 심각 저하
TO-BE · 라벨 whitespace-nowrap + (필요 시) 가로 스크롤 / text-[10px] 자동 축소
원인 추정 · 수익률/자산 + 1개월~전체 두 토글 그룹이 같은 flex row · 각 버튼이 text-[11px] px-2.5 고정, viewport가 좁아지면 단어 자동 줄바꿈

P1 · WARNH-2. 수익률 TOP 종목명 트렁케이션

DIREX DAIL SEMI BU 3X ET-USD   ← SOXL (실제: DIREXION DAILY SEMICONDUCTOR BULL 3X)
PROSHARES ULTRA SEMICONDUCT    ← USD ticker (실제: PROSHARES ULTRA SEMICONDUCTORS)

KIS 종목 마스터의 영문명 길이 제한으로 잘린 채 저장됨. → Yahoo Finance 영문명 fallback 또는 stocks.engName 풀네임 컬럼 추가 권장.

P1 · WARNH-3. 토글 버튼 a11y 누락

"수익률/자산", "1개월/…" 토글 버튼에 aria-pressed 또는 role="tab" aria-selected 없음. 보유 페이지의 계좌 탭은 aria-selected가 적용돼 있어 일관성 위해 동일 패턴 권장.

INFOH-4. "기간 변화: +11.15%p" 표기

최근 커밋(4b8ef7a)에서 일간/주간 변동의 %p를 제거했으나, 기간 변화는 그대로 유지 — 수익률 간 포인트 차이라 정확한 표기이지만 다른 영역과 어긋난다는 점은 인지 필요.

H-1 증거 · 토글이 글자 단위로 세로 분리
"자산" 토글 활성화 후 (정상 전환)

P1 · WARNP-1. 종목 검색 즉시 트리거 부재

"삼성"을 타이핑해도 결과가 즉시 표시되지 않고 Enter 또는 우측 돋보기 버튼을 눌러야 검색 실행. 자동완성 UX와 다르므로 사용자가 "검색이 안 되네?"라 오인할 수 있음.

권장 · onChange debounce(300ms) → 즉시 결과 노출

P1 · WARNP-2. 정렬 토글 a11y (평가금/매입금/수익금)

현재 정렬 기준이 시각만으로 노출. aria-sort 또는 aria-pressed 추가 필요.

P2 · WARNP-3. FAB 두 개가 종목 카드 일부 가림

우하단에 AI 어시스턴트 + 종목 추가 두 FAB(fixed right-4 z-40 w-12 h-12)가 떠 있어 좁은 폭에서 스크롤 시 종목 카드 수익률/금액과 겹침. 카드 우측 내부 패딩 64px 확보 또는 스크롤 시 자동 숨김 권장.

도넛 + 예수금 + 계좌 탭
기본 계좌 자동 선택 동작 확인
인라인 검색 결과 스크롤 정상
