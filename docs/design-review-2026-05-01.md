# Snapshot Finance — 모바일 UI 디자인 리뷰

검토일: 2026-05-01 · 환경: localhost:3000 (모바일 폭 300px, DPR 2 · iOS Pretendard Variable)
검토 화면: `/dashboard`, `/dashboard/snapshots`, `/dashboard/portfolio`, `/dashboard/more`
검토 관점: 타이포 위계, 간격/정렬, 모션, 터치 타깃, safe-area, 다크모드 대비, 정보 위계, 일관성

## 0. 한 줄 요약

전반적인 톤(웜 오프화이트 배경, Snapshot 그린, Pretendard, 레전드-카드 컬러 매칭)은 정돈되어 있어 좋은 출발점이다. 다만 (1) **₩ 글리프가 strikethrough처럼 보이는 폰트 렌더링 결함**, (2) **하단 네비 5탭 비대칭 + 라벨 줄바꿈**, (3) **FAB이 핵심 콘텐츠를 가림**, (4) **safe-area-inset 미적용**, (5) **무변동/상승/하락 색상 시맨틱 모호**, 이 다섯 가지가 신뢰감을 가장 크게 떨어뜨리고 있다. 모두 작은 수정으로 즉시 잡을 수 있다.

## 1. 우선순위 P0 (출시 전 반드시 수정)

### 1-1. ₩ 심볼이 strikethrough처럼 보이는 글리프 충돌

대시보드 메인 헤드라인 `₩88,945,190`, 누적손익 `₩10,824,186`, 스냅샷 카드의 모든 금액, 도넛 가운데 총자산, 종목 리스트 가격 — **앱 전반의 모든 통화 표시**에서 ₩의 가로 두 줄이 인접 숫자에까지 이어진 것처럼 보여 마치 가격 전체에 strikethrough가 그어진 듯 보인다.

**원인.** Pretendard Variable의 ₩(U+20A9) 글리프는 Bold/Extra-Bold 가중치에서 두 가로선이 sidebearing 바깥까지 길게 그려진다. 옆 숫자가 letter-spacing 없이 바로 붙으면 가로선이 시각적으로 이어진다. 사이즈가 클수록(예: `text-5xl/6xl + font-bold`) 두께가 두꺼워져 더 눈에 띈다.

**해결책 (권장 순서).**

1. 통화 단위와 숫자를 분리해 ₩에 우측 여백을 부여한다.

   ```tsx
   // components/ui/Currency.tsx
   export function Currency({ value }: { value: number }) {
     return (
       <span className="tabular-nums">
         <span className="mr-[0.12em] font-medium">₩</span>
         {new Intl.NumberFormat('ko-KR').format(value)}
       </span>
     );
   }
   ```

   - `mr-[0.12em]`로 ₩ 가로선과 숫자 사이를 떨어뜨린다(시각적 충돌 해소).
   - `font-medium`으로 ₩만 가중치를 한 단계 낮추면 가로선 두께가 가늘어져 strikethrough처럼 보이지 않는다.
   - `tabular-nums`로 숫자 폭을 등폭화해 우측 정렬도 안정된다.

2. 또는 Tailwind 글로벌 유틸로:

   ```css
   .num-krw::before {
     content: "₩";
     font-weight: 500;
     margin-right: 0.12em;
   }
   ```

3. 더 강한 해결: `font-feature-settings: "ss01"` 또는 ₩만 Apple SD Gothic Neo로 fallback. (Pretendard에는 ₩ stylistic alternate가 없어서 1·2번이 현실적이다.)

**기대 효과.** 한 곳만 고치면 앱 전 화면의 약 20곳 이상에서 같은 결함이 동시에 사라진다. 가장 ROI 높은 수정이다.

### 1-2. 하단 네비게이션 — 5탭 비대칭 + "포트폴리오" 라벨 줄바꿈

현재 하단 네비는 [홈(N 아바타)] · 포트폴리오 · 스냅샷 · 더보기 · [설정(globe)] 5개. 측정값:

- 각 탭 폭 58px · 높이 47px (5×58=290px이라 viewport 300px에 거의 꽉 참)
- "포트폴리오"는 5글자라 한 줄에 안 들어가 **"포트폴 / 오"로 줄바꿈** → 자체 높이만 57px가 되어 다른 탭(47px)과 시각 정렬이 깨짐
- 좌측 N 아바타는 라운드 솔리드, 우측 globe는 흰 배경 + 그림자 → **두 끝 아이콘이 서로 다른 시각 언어**로 floating chip처럼 보여 5개 탭이 한 그룹으로 안 묶임
- **N 아바타가 무엇을 의미하는지 모호** (홈인지 프로필인지). `aria-label`은 "홈"인데 시각적으로는 사용자 이니셜이라 의미 충돌

**해결책.**

1. 탭 라벨 짧게: "포트폴리오" → "**보유**" 또는 "**자산**"
2. 5탭은 모바일 권장 한계치. 가능하면 4탭으로 줄이고, **설정은 더보기 안으로 흡수** (현재 더보기에 설정 항목이 없는 것도 어색함)
3. 양 끝 floating chip 디자인을 버리고 **5개 탭 모두 동일한 시각 언어**로 통일. Material/HIG 표준 패턴.
4. `홈` 탭의 N 아바타를 일반 home 아이콘으로 교체. 사용자 정보 진입은 헤더 우상단 사람 아이콘으로 충분.
5. 탭에 `aria-current="page"` 부여 (현재 색상만으로 active 표현 — 스크린리더에서 활성 탭 식별 불가)

### 1-3. FAB(AI 어시스턴트)이 콘텐츠를 가림

48×48px FAB이 viewport 우하단(약 x:236, y:522 / 300×650 기준)에 떠 있고 z-index 40. 측정 결과:

- 대시보드: "최근 스냅샷 - 전체보기" 버튼의 sparkle 아이콘과 **충돌해 시각적으로 두 버튼이 겹친 chimera**처럼 보인다
- 포트폴리오: 첫 종목 카드의 등락률(`▲ 4...`)이 가려져 **숫자가 안 보임**
- 스냅샷: 마지막 카드 일부, SOFI 카드의 가격/등락률 마지막 두 자리가 잘림

**해결책.**

1. 콘텐츠 컨테이너에 **`pb-32` 이상의 안전 패딩** 부여(현재 `pb-24` = 96px 정도이지만 FAB 높이+여백을 고려하면 부족). 측정으로는 main에 `pb-24`가 들어 있으나 FAB가 `bottom-?`으로 위치 잡혀 있어 결과적으로 가려짐.
2. FAB 위치를 **하단 네비 위쪽 16px 안전 영역**으로 명시 (`bottom: calc(72px + env(safe-area-inset-bottom) + 16px)`)
3. 또는 FAB을 **scroll-aware**로 만들어 콘텐츠 위에서 숨기고 하단 도달 시 표시 (Material 패턴)
4. 차선책: FAB을 **하단 네비 중앙에 통합** (Instagram·Threads 패턴) — 5개 탭을 4탭으로 줄이는 것과 자연스럽게 결합 가능

### 1-4. safe-area-inset 전무

측정값: `body.padding-bottom: 0px`, 네비 `padding-bottom: 0px`, 어디에도 `env(safe-area-inset-*)` 미사용.

- iOS PWA로 홈 화면에 추가했을 때 home indicator 영역(약 34pt)에 하단 네비 컨텐츠가 가려진다
- 노치/Dynamic Island가 있는 기기에서 헤더가 잘릴 수 있다 (다만 헤더는 `position: static`이라 스크롤이 회피해 줌)

**해결책.**

```css
/* globals.css */
:root { --safe-bottom: env(safe-area-inset-bottom, 0px); --safe-top: env(safe-area-inset-top, 0px); }

nav.bottom-tabs {
  padding-bottom: calc(8px + var(--safe-bottom));
  height: calc(72px + var(--safe-bottom));
}
header { padding-top: calc(14px + var(--safe-top)); }
```

`<meta name="viewport" content="..., viewport-fit=cover">`도 함께 확인.

### 1-5. 등락 색상 시맨틱 모호

- 한국 주식 컨벤션을 따라 **상승=빨강**, **하락=녹색** — OK.
- 그러나 **`destructive` 토큰(`#d4322f`)을 상승 색상으로 재활용** 중. 의미 충돌(스크린리더가 "destructive"로 읽힐 수 있음, 다크모드 토큰을 따로 관리해야 함, 토큰 의미 추적이 불가).
- **무변동(0.00%)도 빨강으로 표시** ("지금과 +0.00% 차이"가 빨강) → 사용자가 "올랐는지 안 올랐는지" 즉시 판별 불가

**해결책.**

1. 등락 전용 시맨틱 토큰 신설:

   ```css
   :root {
     --color-up: #d4322f;   /* 한국 상승 빨강 */
     --color-down: #1e7d4f; /* 한국 하락 녹색 */
     --color-flat: #737373; /* 무변동 회색 */
   }
   .dark {
     --color-up: #f06b68;
     --color-down: #4cc38a;
     --color-flat: #a3a3a3;
   }
   ```

2. `destructive`는 삭제·확인 다이얼로그 등 **파괴적 액션 전용**으로 격리.
3. 0% 표시는 회색 + ▬ 또는 ▪ 마커. `Math.abs(rate) < 0.005` 임계치로 분기.

## 2. 우선순위 P1 (다음 스프린트)

### 2-1. 스냅샷 페이지 헤더 중복

스크롤 영역 상단에 같은 "Snapshot" 로고 + 제목이 두 번 나타난다(앱 헤더 + 페이지 헤더). H1 "스냅샷"만 남기거나, 페이지 헤더의 로고/벨 아이콘을 제거. 벨(알림) 아이콘은 다른 페이지에는 없어 일관성도 깨짐.

### 2-2. 카드 액션 패턴 비일관

- 스냅샷 카드: 텍스트 링크 `상세 | 시뮬레이션 | 삭제` + 우측에 라벨 없는 회색 사각(체크박스로 추정)
- 포트폴리오 카드: 아이콘+텍스트 `✏️ 수정 | 🗑 삭제`
- 두 패턴이 다르고, 스냅샷의 "회색 사각"은 affordance 없음

**제안.**

- 한 패턴으로 통일: **카드 우상단 ⋯ overflow menu** (액션 빈도가 낮은 편이므로). 가장 빈번한 액션 1개만 outline 버튼으로 노출.
- 다중 선택(스냅샷 비교 기능)이 필요하면 **선택 모드 토글**을 페이지 상단에 두고, 모드 진입 시에만 체크박스 표시 (Apple Mail 패턴)
- 매 카드마다 항상 보이는 삭제는 실수 위험 + 시각 노이즈 큼

### 2-3. 포트폴리오 테이블 헤더에 "추가" 버튼 침범

`보유 종목 · 18 / 평가금액↓ / 매입금액 / [+ 추가]` — 컬럼 헤더와 액션 버튼이 한 줄에 섞여 "추가"가 정렬 옵션처럼 보인다. 게다가 "평가금액"은 "평가금/액", "추가"는 "추/가"로 줄바꿈.

**제안.**

- "+ 종목 추가" 버튼은 페이지 상단 H1 우측 또는 하단 FAB 위치로 분리
- 헤더는 컬럼 정렬용 토글로만 (`평가금액 ↓`)
- 좁은 폭에서 "매입금액" 컬럼 숨기고 카드 본문의 보조 메타로 처리 (이미 `평단 $11.90` 식으로 보여주고 있어 중복)

### 2-4. 종목 카드 "평가손익률" 컬럼 위계

현재 상승률만 우측에 크게 보임. 그러나 "21.1%(비중)" 같은 정보가 본문 부제에 있어 혼동:

```
TSLL · 1,018주 · 평단 $11.90 · 21.1%
```

`21.1%`가 비중인지 수익률인지 메타 한 줄에 섞여 즉시 안 읽힌다.

**제안.** 비중에는 라벨 prefix(`비중 21.1%`) 또는 작은 칩(`21.1% 비중`)으로 명시. 또는 카드 좌측 컬러바 하단에 비중 게이지 막대를 그려 시각적으로 분리.

### 2-5. 도넛 차트 레전드는 6개만 (18종목 중)

레전드가 잘려 있어 색상-종목 매칭이 12개 종목에서 불가. "기타 N%" 묶음 또는 레전드 toggleable expand가 필요.

### 2-6. 더보기 페이지가 너무 비어있음

3개 카드만 있고 60% 빈 공간. 사용자 프로필·계정·로그아웃·테마·언어·데이터 백업 등 핵심 설정이 모두 없다. "설정"이 별도 탭으로 분리되어 있어 의미가 더 모호해짐. **"더보기"와 "설정"을 통합**하면서 시뮬레이션·만약에·M7 뉴스는 진짜 "도구(Tools)" 섹션으로 묶고 그 아래 "계정", "환경설정", "정보" 그룹을 추가하는 것을 권장.

### 2-7. "최근 스냅샷" 카드 - "전체보기" 버튼과 sparkle 아이콘 충돌

대시보드 하단의 `최근 스냅샷 [전체보기 ⋆+]` — 둥근 사각 버튼 위에 sparkle 아이콘이 절반 겹쳐 있어 두 컴포넌트가 시각적으로 한 객체처럼 보인다. 별도 액션이라면 간격을 늘리거나, 같은 액션이라면 sparkle을 버튼 안으로 흡수.

## 3. 우선순위 P2 (디테일 정리)

### 3-1. 상단 헤더 터치 타깃 부족

- 테마 토글: **36×36px** (권장 44pt 미달)
- 프로필 아이콘: **32×32px** (권장 44pt 미달, 실패율↑)

투명한 `::before` pseudo-element로 터치 영역만 44px로 확장하거나 패딩 추가.

```tsx
<button className="relative p-2.5 -m-2.5">  {/* 터치 영역 44px 보장 */}
  <SunIcon className="size-5" />
</button>
```

### 3-2. 다크모드 대비

다크모드에서 배경이 거의 순흑(#000)에 가까워 OLED 친화적이지만:

- 카드와 배경 대비가 약함 (둘 다 검정 계열) → 카드 경계가 잘 안 보임
- "전체보기" 그린 버튼의 채도가 낮아져 묻힘
- ₩ strikethrough 문제는 다크모드에서 **더 두드러짐** (밝은 글씨에 가로선이 더 잘 보임)

**제안.**

- 다크 카드 배경을 `#171717` 또는 `#1a1a1a`로 (페이지 배경은 `#0a0a0a` 유지)
- 카드에 1px subtle border (`#ffffff10`)로 분리감 강화
- 그린 토큰을 다크 전용으로 한 단계 밝게 (`#22a06b`)

### 3-3. 헤더가 sticky 아님

`header { position: static }`. 긴 스냅샷/포트폴리오 리스트에서 헤더가 사라져 컨텍스트(앱명·테마·프로필 진입)를 잃는다.

**제안.** `position: sticky; top: 0; backdrop-filter: blur(16px)`로 처리 (하단 네비처럼 일관된 frosted glass).

### 3-4. 카드 모서리·그림자 일관성

대시보드 차트 카드 vs 원금/누적손익 카드 vs 종목 카드의 그림자 강도와 테두리 두께가 미세하게 다르다. shadcn token 정의하고 `Card` 컴포넌트 한 곳에서만 관리.

### 3-5. 모션

- 네비 색상 트랜지션: `color 0.15s cubic-bezier(0.4, 0, 0.2, 1)` — 적절한 마이크로 트랜지션. OK.
- 페이지 라우팅 시 별도 transition 없음 → 모바일 앱 느낌을 살리려면 Next.js + view-transition-api로 push/back 슬라이드 모션 도입 검토 (선택).
- FAB 호버/탭 액티브 스케일 피드백 추가(`active:scale-95`).

### 3-6. 정보 위계 (대시보드)

```
₩88,945,190
▲ 15.22%   +₩10,824,186
```

상승률과 절대 손익이 동일한 weight·size라 우열이 안 보인다. 둘 중 하나(권장: 상승률)를 1.25x 크기 또는 더 진한 색으로.

### 3-7. 날짜 포맷 비일관

- 대시보드: `2026.05.01 · 오후`
- 스냅샷 카드: `2026-05-01 / 07:30 · 18종목`
- 차트 축: `03.26`, `05.01`

세 곳이 모두 다른 구분자(`.`, `-`). 한 가지로 통일 (Pretendard에서는 점 구분이 시각적으로 더 깔끔함 → `2026.05.01`).

### 3-8. "Auto Snapshot" / 수동 스냅샷 라벨

스냅샷 페이지에서 LATEST 카드만 "Auto Snapshot" 라벨이 보이고 나머지는 라벨이 없어 자동/수동 구분 불가. 모든 카드에 작은 chip(자동/수동)을 붙이거나, 자동만 점·아이콘으로 표시.

### 3-9. 지수 표기 한국어 일관성

`Auto Snapshot`(영어), `LATEST · 최신`(영문 strap + 한글), `TIMELINE · 타임라인` — 영문 strap 패턴이 멋있어 보이지만 일부만 적용. 전부 적용하든지 전부 한글로 통일.

### 3-10. 빈 카드 클릭 영역

스냅샷 타임라인 카드 전체가 클릭 가능한 것처럼 보이지만 실제로는 "상세" 텍스트만 클릭 가능. 카드 전체를 클릭 가능 영역으로 확장하면 모바일 사용성이 크게 좋아진다 (`<Link>`로 카드 wrap).
